Domínio

É um assunto para qual o software está sendo usado

É uma atividade executada pelo usuário dentro do software

Domínio

Deve ser independente de serviços externos, técnologias, frameworks, etc.

Sub-Domínio

Um domínio pode ser granulado em sub-domínios

Domínio Principal (core-domain)

É a principal parte do domínio

Especialistas de Domínio

Não é um cargo, é uma role

Modelo

É uma simplificação. É uma interpretação da realidade que abstrai os aspectos relevantes para resolver o problema em questão e ignorar detalhes alheios a isto.

Modelo

É evolutivo

Serve como algo comum e palpável a todos os membros do time

Garante que aquilo que está sendo especificado é o que está sendo implementado

É o conhecimento destilado

A implementação DEVE refletir 100% o modelo

Exemplos de Modelos

Banco imobiliário é um modelo que abstrai o domínio do mercado imobiliário

Maquetes são modelos do domínio de arquiteturas de edificações
